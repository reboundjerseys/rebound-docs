# Data Flow

## Browse and search

```mermaid
flowchart LR
  Client[Web or mobile client] --> Backend[Backend API]
  Backend --> Search[Algolia]
  Search --> Backend
  Backend --> Client
```

## Create a listing

```mermaid
flowchart LR
  Client[Web or mobile client] --> Backend[Backend API]
  Backend --> DB[(Supabase)]
  Backend --> Search[Algolia]
  Backend --> External[Stripe / eBay / notifications]
```

## Image analysis

```mermaid
flowchart LR
  Upload[Upload image] --> Storage[(Supabase Storage)]
  Storage --> Backend[Backend API]
  Backend --> Scanner[AI Jersey Scanner]
  Scanner --> Backend
  Backend --> DB[(Supabase)]
```

## Authenticity check

```mermaid
flowchart LR
  Image[Input image] --> Auth[Authentication service]
  Auth --> Pinecone[(Pinecone)]
  Pinecone --> Auth
  Auth --> Verdict[Verdict and confidence]
```

The platform stores the returned authenticity state in product and history records after the service responds.

## Why this matters

These flows make the platform easier to reason about:

- clients submit data
- the backend coordinates work
- persistence happens in Supabase
- search and AI systems derive secondary data
