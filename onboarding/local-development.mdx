# Local Development

## Docs site

Run the Mintlify preview from the root of this repository:

```bash
mint dev
```

Use `mint broken-links` before you publish doc updates.

## Platform repos

Each product repository owns its own development commands.
Use the repository-specific README and package scripts instead of guessing.

## Working rule

Read the inventory first, then the code, then the docs.
That order keeps the docs aligned with the source of truth.
