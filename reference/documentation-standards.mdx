# Documentation Standards

## Page shape

Each page should answer:

- why the system exists
- what it owns
- how it communicates
- how developers interact with it

## Page sources

Use this order of truth:

1. inventory
2. current source code
3. existing docs
4. README files

## Placeholder rule

If a system lives in another repository, create a placeholder page here.
Do not speculate.

## Validation rule

- Update navigation when adding a page.
- Run `mint broken-links`.
- Keep related pages linked.
