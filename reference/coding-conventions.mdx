# Coding Conventions

## Write for engineers

- Use active voice.
- Use second person when helpful.
- Keep one idea per sentence.

## Keep the source of truth clear

- Prefer the inventory and code over memory.
- Do not invent behavior.
- Mark uncertainty as `TODO: Needs Investigation`.

## Keep docs navigable

- Use system-oriented pages.
- Cross-link instead of duplicating.
- Use code formatting for file names, paths, and commands.

## Keep terminology stable

- Use the glossary terms.
- Keep "listing", "product", and "order" distinct.
